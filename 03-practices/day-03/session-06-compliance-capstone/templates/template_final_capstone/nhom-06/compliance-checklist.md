# Compliance Checklist

## Đánh giá tiêu chí bảo mật

| Tiêu chí | Trạng thái |
|---|---|
| Không dùng dữ liệu thật | Đạt |
| Không dùng thông tin khách hàng thật | Đạt |
| Không dùng IP thật, mã trạm thật hoặc cấu hình thật | Đạt |
| Có dữ liệu mô phỏng thay thế | Đạt |
| Có mô tả nguồn dữ liệu mô phỏng | Đạt |
| Không dùng token thật | Đạt |
| Không dùng API key thật | Đạt |
| Không hard‑code thông tin bí mật | Đạt |
| Không đưa API key hoặc token vào prompt, ảnh chụp màn hình, log, n8n export hoặc bài nộp | Đạt |
| Nếu mô tả điểm kết nối (endpoint), chỉ dùng endpoint giả lập | Đạt |
| Quyền truy cập trong bài thực hành là quyền đọc hoặc quyền mô phỏng | Đạt |
| Có bước con người trong vòng lặp (human in the loop) | Đạt |
| Có người chịu trách nhiệm duyệt đầu ra | Đạt |
| Có tiêu chí duyệt hoặc từ chối | Đạt |
| Không để AI tự quyết định việc có rủi ro kỹ thuật, pháp lý, nhân sự hoặc vận hành | Đạt |
| Có lưu đầu vào mẫu | Đạt |
| Có lưu đầu ra mẫu | Đạt |
| Có ghi phiên bản prompt hoặc hướng dẫn | Đạt |
| Có ghi trạng thái thành công hoặc lỗi | Đạt |
| Có cách truy lại lý do AI đưa ra kết quả | Đạt |
| Nhật ký vận hành chỉ chứa dữ liệu phi nhạy cảm | Đạt |
| Có quy tắc từ chối nếu câu hỏi vượt phạm vi | Đạt |
| Có quy tắc không suy đoán khi thiếu dữ liệu | Đạt |
| Có quy tắc yêu cầu trích dẫn hoặc nêu căn cứ khi dùng tài liệu | Đạt |
| Có quy tắc chuyển sang con người khi rủi ro cao | Đạt |

*Ngày cập nhật: tháng 7/2026*
